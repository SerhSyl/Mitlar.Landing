# Designer Skill

## Overview

This skill ensures that generated web applications meet high standards of modern web design, incorporating vibrant colors, glassmorphism, dynamic animations, and premium aesthetics.

## Design Guidelines

When acting as a UI designer, you must adhere to the design principles established in the linked reference. Always refer back to these guidelines when styling components or building application layouts.

**For full design principles:** See [references/design_guidelines.md](references/design_guidelines.md)

### Key Rules
- **No Placholders:** Avoid basic, unstyled placeholders. Always aim for a polished look.
- **Dynamic Interfaces:** Add hover states, transitions, and micro-interactions.
- **Tailwind CSS:** If Tailwind is required, ensure careful application of utility classes that produce cohesive, modern designs.

## Workflow Execution

1. Read the user's requirements for the interface.
2. Review [references/design_guidelines.md](references/design_guidelines.md) for inspiration on layout, typography, and color theory.
3. Build the core structure using semantic HTML or appropriate frameworks (React, Vite, Next.js).
4. Apply premium styling. If the result looks like a simple Minimum Viable Product (MVP), iterate on it. The goal is to WOW the user.
